# 💻 Sistema de Gestión de Equipos - Foro 2 POO

Este proyecto ha sido desarrollado como parte de la Experiencia de Aprendizaje 1 de la materia **Programación Orientada a Objetos** en la **Universidad Don Bosco (UDB)**. El objetivo es aplicar los fundamentos de la POO y el uso de interfaces gráficas básicas en Java.

## 🚀 Características del Proyecto
* **Paradigma POO:** Uso estricto de Clases, Objetos, Encapsulamiento y Herencia.
* **Jerarquía de Clases:** Implementación de una clase base `Equipo` y subclases para `Desktop`, `Laptop` y `Tablet`.
* **Interfaz Gráfica (GUI):** Interacción amigable mediante `JOptionPane` de la librería Swing.
* **Persistencia en Memoria:** Manejo de colecciones dinámicas con `ArrayList`.
* **Validaciones:** Control de errores y excepciones para asegurar la integridad de los datos.

## 🛠️ Estructura del Código
El proyecto es modular y se divide en los siguientes archivos:
* `InterfazUsuario.java`: Maneja toda la capa visual y entrada de datos.
* `Equipo.java`: Clase padre con atributos comunes.
* `Desktop.java`, `Laptop.java`, `Tablet.java`: Clases hijas con atributos específicos.
* `GestionDatos.java`: Lógica para el almacenamiento y reporte de la lista.
* `Main.java`: Punto de entrada que coordina el flujo del programa.
